Module PyEnv.DataStack

In Python, everything is an object, and the interpreter maintains a stack of references to such objects. Pushing and popping on the stack are always references to objets that leave in a heap. There is no need to model this heap, but the data stack is quite important.

type cell =
  1. | Const of int
    (*

    index in co_consts

    *)
  2. | Name of {
    1. global : bool;
    2. ndx : int;
    }
    (*

    reference to a name, stored in co_names.

    *)
  3. | VarName of int
    (*

    reference to a local name, stored in co_varnames

    *)
  4. | Temp of T.Ident.t
    (*

    SSA variable

    *)
  5. | Code of {
    1. fun_or_class : bool;
    2. code_name : string;
    3. code : FFI.Code.t;
    }
    (*

    code Python object with its name. It can be a function, class, closure, ...

    *)
  6. | Map of (string * cell) list
    (*

    Light encoding of raw Python tuples/dicts. Only used for type annotations at the moment.

    *)
  7. | BuiltinBuildClass
    (*

    see Python's LOAD_BUILD_CLASS

    *)
  8. | Import of {
    1. import_path : string;
    2. symbols : string list;
    }
    (*

    imported module path, with optional name of symbols

    *)
  9. | ImportCall of T.qualified_procname
    (*

    Static call to export definition

    *)
